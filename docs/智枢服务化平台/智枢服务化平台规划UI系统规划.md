# 餐饮管理系统 UI 设计规划文档

## 一、设计系统总览

### 1.1 设计理念

- **稳重大气** - 深色系 + 精致排版，传达专业感
- **高效清晰** - 层级分明，信息组织合理
- **品质感** - 适当留白，精细细节处理
- **可用性优先** - 关键信息突出，操作流畅

### 1.2 目标用户

- 餐饮店长/总经理（30-55岁）
- 财务/成本管理人员（25-45岁）
- 采购/库存管理人员（22-40岁）
- 需要：快速决策、高效工作、可信的数据呈现

## 七、交互设计规范

### 鼠标悬停效果

- 卡片：背景色加深 2%，投影增加
- 按钮：背景色变亮/加深 8%，鼠标变手指
- 链接：下划线出现，颜色变亮

### 加载态

- 骨架屏用于数据加载
- 微动画旋转圆圈用于操作中
- 进度条用于文件上传

### 反馈设计

- 操作成功：绿色 toast 提示 2s
- 操作失败：红色 toast 提示 3s
- 确认操作：模态框 + 取消/确认按钮
- 数据保存：自动保存 + "已保存" 提示

### 响应式设计

- **Desktop (≥1280px)**：完整布局，3列卡片
- **Tablet (768px-1279px)**：侧边栏可折叠，2列卡片
- **Mobile (<768px)**：底部导航，堆栈布局

---

## 八、可访问性规范

- 所有可交互元素：最小 44px × 44px
- 色彩对比度：至少 4.5:1（文字） / 3:1（图形）
- 键盘导航：完全支持 Tab/Enter/Esc
- 屏幕阅读器：ARIA 标签完整
- 焦点指示：清晰的焦点环（2px）

- **图标**：Heroicons v2

- **类型**：TypeScript 5.3
- **状态**：Pinia
