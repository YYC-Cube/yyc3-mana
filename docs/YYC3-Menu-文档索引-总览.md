## 🎯 使用建议

### 1. 文档命名规范

所有新文档应遵循命名规范：

```
格式: YYC3-Menu-[分类]-[具体名称].md

分类选项:
- 架构设计
- 实施总结
- 开发规范
- 部署运维
- 需求规划
- 审核分析
- 团队规范
- 业务管理
```

### 2. 文档编写标准

参考文档模板：

- [YYC3-实施总结文档模版.md](YYC3-Mune-文档规范/YYC3-实施总结文档模版.md)

必须包含：

- YAML前置数据（元信息）
- YYC³品牌标识
- "五高五标五化"框架
- 结构化内容章节

### 3. 文档维护流程

```yaml
维护流程:
  创建文档:
    1. 选择合适的分类
    2. 按规范命名文件
    3. 使用标准模板
    4. 填写YAML元信息
    
  更新文档:
    1. 修改@updated字段
    2. 记录变更内容
    3. 更新版本号
    4. 提交Git记录
    
  归档文档:
    1. 标记@status为"已归档"
    2. 移动到archive/目录
    3. 更新本索引文档
```
